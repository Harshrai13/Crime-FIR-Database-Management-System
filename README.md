Crime & FIR Database Management System

A database-driven web application designed to manage FIR records, complaint details, officer assignments, and crime categories using MySQL. This system demonstrates strong relational database design, structured SQL queries, and backend integration using Python.


Project Overview

The Crime & FIR Database Management System is built to efficiently manage and track crime records in a structured format. It focuses on database normalization, foreign key relationships, and optimized SQL queries for reliable data handling.

The system allows:
- FIR registration
- Complaint record storage
- Officer assignment
- Case status tracking
- Structured data retrieval using multi-table JOIN queries



Technologies Used

- MySQL (Relational Database)
- Python (Backend Logic)
- HTML
- CSS
- Flask (Lightweight Framework)



Database Features

- Normalized relational schema
- Multiple tables with foreign key constraints
- One-to-many relationships
- Multi-table JOIN queries
- GROUP BY and aggregate functions
- Data integrity enforcement

 
 Main Tables:
- Officers
- Complainant
- Crime_Category
- FIR


 Key Functionalities

- Add new FIR records
- Store complainant details
- Assign officers to cases
- Update case status
- Display structured crime records
- Efficient SQL-based data retrieval
